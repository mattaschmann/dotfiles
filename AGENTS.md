# Dotfiles

Personal dotfiles managed by DotBot. Cross-platform: macOS and WSL2/Linux.

## Install

- `./install` runs DotBot via `install.conf.yaml` (symlinks + shell steps).
- Add new symlinks to `install.conf.yaml` — never symlink by hand.
- `os_specific_links.sh` handles Darwin vs WSL2 links (alacritty, urlview).

## Packages

- `packages.toml` is the single source of truth (apt/brew/cargo/uv/npm/vscode/opencode).
- `Brewfile` is generated by `scripts/generate-brewfile.sh` and gitignored — never edit or commit it.
- `scripts/update.sh` installs/updates everything from `packages.toml`.
- `scripts/cleanup.sh` reports drift (installed but untracked).

## Layout

- Config dirs: `nvim/`, `zsh/`, `tmux/`, `alacritty/`, `starship/`, `tig/`, `rg/`.
- `dotbot/` — submodule, don't edit.
- `scripts/` — install/update/maintenance helpers.
- `opencode/` — opencode global config, symlinked to `~/.config/opencode/`.
- `.tasks/` — task files.

## Conventions

- Lua formatted with stylua (`.stylua.toml`): 2-space indent, single quotes.
- Don't edit symlinked targets in `$HOME` — edit the source here.
