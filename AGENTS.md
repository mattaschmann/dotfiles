# Dotfiles

Personal dotfiles managed by `pkgctl`. Cross-platform: macOS and WSL2/Linux.

## Install

- `pkgctl install` is the single entry point: checks prerequisites, applies symlinks, runs setup steps, updates packages.
- Add new symlinks to `[install.links]` in `packages.toml` — never symlink by hand.
- OS-variant files use `{os}` pattern in paths (e.g. `alacritty/alacritty.{os}.toml`) — pkgctl infers which platforms have variants from files on disk.

## Packages

- `packages.toml` is the single source of truth (apt/brew/cargo/uv/npm/vscode/opencode/install).
- `Brewfile` is generated by `pkgctl brewfile` and gitignored — never edit or commit it.
- `pkgctl` is the CLI tool for all operations.
- Run via: `uv run pkgctl <command>` or `./bootstrap.sh <command>`.

### pkgctl commands

| Command | Purpose |
|---------|---------|
| `pkgctl install` | Full setup: doctor + symlinks + shell steps + copies + package updates |
| `pkgctl update [MANAGER...]` | Upgrade + ensure declared packages installed |
| `pkgctl drift [MANAGER...]` | Report untracked/missing packages (exit 2 if drift) |
| `pkgctl diff [MANAGER...]` | Show pending upstream changes (opencode) |
| `pkgctl list [MANAGER]` | Print what's declared in packages.toml |
| `pkgctl link` | Apply symlinks only |
| `pkgctl clean` | Remove dead symlinks |
| `pkgctl doctor` | Check prerequisites |
| `pkgctl brewfile` | Generate Brewfile from packages.toml |

Flags: `--json` (structured output), `--dry-run`, `--packages-file PATH`.

Managers: apt, brew, cargo, uv, npm, opencode, vscode.

## Layout

- Config dirs: `nvim/`, `zsh/`, `tmux/`, `alacritty/`, `starship/`, `tig/`, `rg/`.
- `ai/` — shared cross-client AI configuration (global instructions + Agent Skills), symlinked to `~/.config/opencode/`, `~/.claude/`, and `~/.kiro/`.
- `opencode/` — OpenCode-specific config (agents, commands, opencode.jsonc), symlinked to `~/.config/opencode/`.
- `src/pkgctl/` — Python package (3.13 + click + dotbot), the CLI.
- `scripts/` — legacy helpers (deprecated, use pkgctl instead).
- `.tasks/` — task files.
- `bootstrap.sh` — ensures uv is installed, then proxies to pkgctl.
- `pyproject.toml` — uv project definition for pkgctl.

## Bootstrap (new machine)

1. Clone this repo.
2. Run `./bootstrap.sh install` — installs uv if missing, then runs full setup.

## Conventions

- Lua formatted with stylua (`.stylua.toml`): 2-space indent, single quotes.
- Don't edit symlinked targets in `$HOME` — edit the source here.
- Python in `src/pkgctl/` uses Python 3.13, formatted with ruff.
