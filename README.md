## Package management
All packages across managers (apt, brew, cargo, uv, npm, vscode) are declared in
[`packages.toml`](packages.toml), the single source of truth. `scripts/update.sh`
(run via `./install`) reads it to generate the `Brewfile` and install/update
everything. Do not edit `Brewfile` directly — it is generated by
`scripts/generate-brewfile.sh`.

- `scripts/update.sh` — install/update all packages from `packages.toml`
- `scripts/cleanup.sh` — detect drift (installed but untracked packages)

## Weekly update
Run the weekly system update routine (packages, zsh plugins via `antidote`, tmux
plugins via TPM) through the `weekly-update` opencode skill, which wraps the
scripts above with interactive cleanup and error debugging. Just ask opencode to
run the weekly update.
